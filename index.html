প্রশ্ন ১/ লোকাল স্টোরেজ এবং সেশন স্টোরেজের পার্থক্য কী?

লোকাল স্টোরেজ এবং সেশন স্টোরেজের পার্থক্যঃ লোকাল স্টোরেজ এবং সেশন স্টোরেজের কাজ হচ্ছে ইউজারের ডাটা ব্রাউজারে স্টোর করে রাখা।যাতে ইউজার যেকোনো সময় ব্রাউজারে আসলে আগের ডাটা গুলো খুঁজে পায়।লোকাল স্টোরেজে ডাটা গুলো স্তায়ীভাবে সংরক্ষন থাকে আমরা ডাটা ডিলিট করার আগ পর্যন্ত।অন্য দিকে সেশন স্টোরেজে ডাটা গুলো অস্থায়ীভাবে সংরক্ষণ থাকে।আমরা ব্রাউজার থেকে বেরিয়ে গেলে বা বা লোড বা রিফ্রেশ করলে ডাটা ডিলেট বা অটো ভাবে রিমুভ হয়ে যায়।
লোকাল স্টোরেজ এবং সেশন স্টোরেজে ৪টি পদ্ধতির মাধ্যমে ইউজারের সাথে ডাটা আদানপ্রদান করে।
১/নতুন ডাটা সেট করাঃsetItem()দ্বারা ইউজার স্টোরেজে নতুন ডাটা সেট করে।
২/স্টোরেজ থেকে ইউজার ডাটা নেওয়াঃস্টোরেজ থেকে ইউজার ডাটা নেওয়ার জন্য getItem()ব্যাবহার করে।
৩/ডাটা রিমুভ করাঃইউজার স্টোরেজ থেকে নির্দিষ্ট ডাটা রিমুভ করার জন্য removeItem() ব্যবহার করে।
৮/স্টরেজ ক্লিয়ার করা:স্টোরেজ ক্লিয়ার করার জন্য. Clear() মেথড ব্যাবহার করে।

প্রশ্ন ২/ইভেন্ট লুপ এর কাজ কী?

ইভেন্ট লুপ এর কাজঃ আমরা জানি জাভাস্ক্রিপ্ট v8 ইঞ্জিন দ্বারা কাজ করে।আমরা জাভাস্ক্রিপ্ট এ যে কোড গুলো লিখি v8 ইঞ্জিন সেগুলোকে মেশিন ল্যাংগুয়েজে রুপান্তরিত করে।ইঞ্জিনের ভেতরে প্রধানত দুটো জিনিস থাকে।হিপ এবং কলস্টেক।হিপের কাজ হচ্ছে মেমরি ম্যানেজমেন্ট করা।
কল স্টেক এর কাজ আমরা যখন জাভাস্ক্রিপ্ট এ ফাংশনগুলোকে কল করি তখন ফাংশন গুলো কল স্টেকে স্টেক আকারে জমা হয়। কলস্টেক থেক কল হবার পর তা ব্রাউজার প্রিন্ট করে দেখায়। কিন্তু যখন কোনো একটা এসিনক্রোনাস ফাংশন কলস্টেকে এসে জমা হয় তখন কল স্টেক এসিংক্রোনাস ফাংশনটিকে কল না দিয়ে তার স্টেক থেকে বের করে web api তে দিয়ে দেয়। web api তে টাইমার কাউন্ট শেষ হবার পর web api ফাংশনটিকে call back queue তে স্তানান্তর করে।call back queue তখন ইভেন্ট লুপ এর মাধ্যামে পর্যায়ক্রমিকভাবে প্রস্তুত ফাংশনগুলোকে কল স্টেকে পাঠাতে থাকে।কল স্টেক থেকে তখন ফাংশন গুলোকে কল করে দিলে তা প্রিন্ট হয়। ইভেন্ট লুপের মাধ্যমে call back queue থেকে ফাংশনগুলো call stack এ স্থানান্তর হয়।
